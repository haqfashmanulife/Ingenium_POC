Turn caching ON
Tells PageServer to cache HTML page templates in memory to improve performance.
Details >>


<< Hide Details
When caching is turned off, the templates must be loaded every time the page is displayed. Caching the templates in memory improves the performance of the PageServer but means that changes to the templates are not visible until either the PageServer is restarted or the PageServer cache is cleared using the control panel.
Turn caching OFF
Tells PageServer to stop caching HTML page templates in memory.
Details >>

<< Hide Details
This is useful when templates are being developed and tested as changes made to the templates are visible the next time the page is displayed. Performance is impacted slightly as the template is loaded every time it is needed. This is not recommended in a production environment.
Caching status
Indicates whether HTML page templates caching is on or off.

CodeServer Control

Clear CodeServer's caches
Clears the caches of allowable values maintained by all caching CodeServer data sources.
Details >>
<< Hide Details

Clearing the CodeServer cache deletes the caches maintained by caching CodeServer data sources, with base INGENIUM these are: Xtab, Edit, Ctlc and DataModel. CodeServer then instructs those data sources that are configured to load the cache at startup to reload their cache.

The process could take an extended period of time since CodeServer needs to retrieve a large amount of data through the CICS gateway transaction and backend database. The time it takes will depend on the CICS gateway speed, the network connection speed and the backend processing speed.
